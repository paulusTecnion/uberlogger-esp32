Overview

Within the control panel you can:

  • Plot measured data from a live stream using the "View live data"-panel;
  • Start SD-card logging using the "Logging"-panel;
  • Configure the logger, input channels and WiFi-functionality through the "Configuration"-panel.

Click on the corresponding buttons above to navigate.

Note: it is essential to configure the logger and its input channels before starting any measurements to ensure the sampled data is accurate.

Quick user manual

Channel configuration

The UberLogger can measure three types of signals: temperature, analog voltage, and digital signals. Additionally, the sample rate, resolution and (analog) scaling can be adjusted to ensure your measurements are recorded accurately.

NTCx/AINx selection: The temperature and analog voltage channels are shared, which means you have to select whether you want to measure temperature or voltage for each individual channel (NTC0/AIN0, NTC1/AIN1, ...). Important: this selection needs to be done on two locations: using the hardware DIP-switches on the UberLogger, and you need to tell the logger what the current DIP-switch configuration is through the "Configuration"-panel.

AINx range selection: The AINx channels can be individually set on two ranges: -10V ... +10V and -60V ... +60V. The range select is changing the analog frontend gain. This results in more bits per Volt when the lower range is used, improving resolution and reducing quantization noise. Important: this selection needs to be done on two locations: using the hardware DIP-switches on the UberLogger, and you need to tell the logger what the current DIP-switch configuration is through the "Configuration"-panel.

Sampling

The analog frontend of the UberLogger has a configurable sample rate and resolution. Before starting any measurement, it is important to select a suitable sample rate and resolution.

Sample rate: The sample rate determines two things: (1) the maximum frequency that can be recorded, and (2) the required SD-card space for a recording. The higher the sample rate, the larger the file that will be written to the SD-card. It is therefore recommended to pick a sample rate that is at approximately 10x ... 20x higher than the highest frequency that needs to be captured, but not higher than that to save disk space. If increased precision is required, a higher factor can be selected, compromising disk space for measurement precision. The UberLogger contains an anti-aliassing filter which removes high-frequency content to reduce aliasing effects.

Resolution: The sampling resolution is adjustable between 12 and 16 bits. Selecting 12 bits disables oversampling and thereby bypasses the digital anti-aliassing filter. This allows measurements which rely on aliassing (frequency-folding) to occur. Note that the hardware anti-aliassing filter is always enabled and has a first-order roll off and a bandwith of 10 kHz.

Time synchronization

The UberLogger has an integrated real-time clock (RTC) which accurately keeps track of the elapsed time. The RTC can be synchronized to your PC's time easily through the "Configuration"-panel, by clicking on "Synchronize to local time". The data timestamp in the live data view will update to the local time. Important: UberLogger will save all data to the SD-card in the coordinated universal time (UTC) format, which excludes any time zone offset. This format allows effortless merging of multiple recorded datasets and gives accurate reader-independent results.
