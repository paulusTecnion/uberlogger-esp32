Quick user manual

Overview

The UberLogger control panel allows you to manage and monitor your logging device. The key functionalities include:

  • Plotting live data streams via the "Live data" page
  • Starting and stopping SD-card logging via the "Logging" page
  • Configuring logger settings, input channels, sample rate, resolution, and WiFi functionality via the "Configuration" page

Note: It is essential to configure the logger and its input channels correctly before starting any measurements to ensure accurate data.

WiFi configuration

How to configure WiFi

The UberLogger can operate in two modes:

  1. Hotspot mode: Connect directly to the UberLogger via a device with WiFi. Access the control panel via http://192.168.4.1.
  2. Client + Hotspot mode: Configure the logger to connect to a WiFi network. Access the control panel via the IP address assigned through the DHCP server.

You can configure the WiFi settings in the "Configuration" page under the "WiFi Configuration" section.

Note: When connecting to your access point, only 2.4 GHz Wi-Fi WPA2/WPA3 networks are supported.

Channel configuration

Setting analog inputs

The UberLogger can measure temperature, analog voltage, and digital signals. Ensure the correct configuration for accurate results:

  • Enabling channels: You can enable or disable or signal using the enable checkbox. Disabling channels leads to smaller log files. Ideal for long measurements.
  • Custom labels: You can give each label a custom name. The maximum length is 16 characters and cannot contain any space or special characters, except hyphens '-' and underscores '_'.
  • NTCx/AINx selection: Choose between temperature or voltage for each channel using both the hardware DIP-switch and the configuration page.
  • AINx range selection: Adjust the input voltage range (-10V ... +10V or -60V ... +60V) for better resolution and reduced noise. This should also be set using the DIP-switch and the configuration page.

Sampling settings

Choosing the right sample rate

The UberLogger allows configurable sampling rates and resolutions to capture data accurately. The sample rate affects:

  • Maximum frequency: The higher the sample rate, the higher the frequency the logger can capture.
  • Storage space: Higher sample rates require more space on the SD card.
  • Frequency requirements: The appropriate sampling frequency for your measurements depends on the signal you wish to capture accurately. It is recommended to set the sample rate at least twice the maximum frequency of the signal of interest. For instance, if your signal's highest frequency is 50 Hz, logging at 100 Hz will prevent aliasing (signal distortion due to a too low sample rate). For temperature logging, most applications can achieve satisfactory results with sampling frequencies as low as 1 Hz or 5 Hz. This ensures sufficient data points for precise temperature measurements without significantly increasing data storage requirements.

The resolution can be set to 12-bit or 16-bit. Higher resolutions improve accuracy but may impact performance at high sample rates.

Averaging

How averaging works

For sample frequencies below 1 Hz, you can choose to average the samples over the desired sample frequency or only take one sample at the desired sample rate. For example, if you enable this option with a sample frequency of 1 Sample/minute, the average will be taken over 1 minute of samples using an IIR filter. After 1 minute, the average will be output to the file.

Note: Averaging is performed using a 25 Hz sample rate and an IIR low-pass filter with a -3 dB cut-off frequency at the desired output sample rate.

Measurement Mode

Setting continuous measurement or external trigger/control

The UberLogger supports three measurement modes:

  • Continuous measurement: Start and stop logging via the web interface or the physical button.
  • External trigger: The logger waits for an external trigger signal to start logging. Note: Live view is not working when no trigger has trigged a measurement.
  • External control: Start/stop logging is fully controlled via a digital input. Note: The physical start / stop button and the web interface start / stop buttons are not working in external controller mode.

For external trigger and external control, you can also set the trigger debounce time to stabilize the trigger input, reducing noise in trigger signals. By default this is set to 100 ms. But you can also increase it up to 60000 ms to create a delayed measurement. The debouncing timer will reset upon getting to low level within the debounce time, restarting the debounce timer.

Time synchronization

Synchronize the Uberlogger time with your local time

The UberLogger features an internal real-time clock (RTC) that tracks time, even during power off (if the battery is installed, which it is by default). You can synchronize the RTC with your local time via the "Configuration" page. This ensures accurate timestamps for recorded data.

Note: Data is saved in Coordinated Universal Time (UTC), which helps merge datasets easily across time zones.

File options

Customize file names and size

The File Options section allows you to configure how the UberLogger saves its data.

File Format:

  • CSV: CSV format writes data directly to CSV files, ideal for low sample rates or short measurement durations. CSV files are easily readable in tools like Excel.
  • Raw Data: Raw format records raw data for long measurements, which can be later converted into CSV files using the provided Python or Windows conversion tools. Raw data is more storage efficient for long recordings.

File Name Mode:

  • Sequential: Generates filenames in a sequential order (e.g., `log0`, `log1`, etc.), overwriting existing files with the same name.
  • Timestamp: Generates filenames with a timestamp in the format `YYYYMMDD_Time_FilePrefix.csv`. This avoids overwriting and helps track when files were created.

File name and split size:

  • Prefix: Set your own prefix for a filename. No special characters can be used and maximum of 70 length characters is allowed.
  • File split size: automatically split files into smaller files. Configurable with kB, MB, or GB sizes. Maximum split size 4 GB.

Calibration

Zero calibrating your Uberlogger

The Calibration section allows you to perform zero-calibration of the analog channels.

Zero calibration ensures that each analog input reads 0 when no signal is present. It is essential for accurate measurements.

You can initiate calibration by pressing the "Start Calibration" button. Ensure that all analog and temperature inputs are all disconnected and all channels are configured as analog inputs both with the DIP switches as well in on the Configuration page before starting calibration, to get accurate results.

Import/Export Configuration

Back and restore settings

The Import/Export Configuration section allows you to save or load the configuration settings of the UberLogger.

Export Configuration: You can download the current configuration settings to a JSON file for backup or to apply to another device later.

Import Configuration: This allows you to load a previously saved JSON configuration file to restore settings or replicate configurations across multiple devices.

File Browser

Browsing through files

The File Browser allows you to manage the data files stored on the SD card.

You can download, delete, or refresh the list of files. Formatting the SD card and unmounting it are only possible when the logger is not actively logging data.

  • Download: Click on "Download" to retrieve a file from the SD card to your local machine.
  • Delete: Select "Delete" to remove the file from the SD card.
  • Format SD-card: This option will erase all contents from the SD card and prepare it for fresh data logging.
  • Unmount SD-card: Safely unmount the SD card to remove it without causing data corruption.
