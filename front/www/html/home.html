Quick user manual

Overview

The UberLogger control panel allows you to manage and monitor your logging device. The key functionalities include:

  • Plotting live data streams via the "Live data" page
  • Starting and stopping SD-card logging via the "Logging" page
  • Configuring logger settings, input channels, sample rate, resolution, and WiFi functionality via the "Configuration" page

Note: It is essential to configure the logger and its input channels correctly before starting any measurements to ensure accurate data.

WiFi configuration

How to configure WiFi

The UberLogger can operate in two modes:

  1. Hotspot mode: Connect directly to the UberLogger via a device with WiFi. Access the control panel via http://192.168.4.1.
  2. Client + Hotspot mode: Configure the logger to connect to a WiFi network. Access the control panel via the IP address assigned through the DHCP server.

You can configure the WiFi settings in the "Configuration" page under the "WiFi Configuration" section.

Channel configuration

Channel settings explanation

The UberLogger can measure temperature, analog voltage, and digital signals. Ensure the correct configuration for accurate results:

  • NTCx/AINx selection: Choose between temperature or voltage for each channel using both the hardware DIP-switch and the configuration page.
  • AINx range selection: Adjust the input voltage range (-10V ... +10V or -60V ... +60V) for better resolution and reduced noise. This should also be set using the DIP-switch and the configuration page.

Sampling settings

Sampling settings and importance

The UberLogger allows configurable sampling rates and resolutions to capture data accurately. The sample rate affects:

  • Maximum frequency: The higher the sample rate, the higher the frequency the logger can capture.
  • Storage space: Higher sample rates require more space on the SD card.

The resolution can be set to 12-bit or 16-bit. Higher resolutions improve accuracy but may impact performance at high sample rates.

Averaging

How averaging works

For sample rates below 1 Hz, the UberLogger can average data samples. This is done using an IIR filter. At 1 Hz or lower, the filter outputs an averaged value every minute.

Note: Averaging is performed using a 25 Hz sample rate and an IIR low-pass filter with a -3 dB cut-off frequency.

Measurement Mode

Measurement modes overview

The UberLogger supports three measurement modes:

  • Continuous measurement: Start and stop logging via the web interface or the physical button.
  • External trigger: The logger waits for an external trigger signal to start logging.
  • External control: Start/stop logging is fully controlled via a digital input.

For external trigger and external control, you can also set the trigger debounce time to stabilize the trigger input, reducing noise in trigger signals.

Time synchronization

RTC Synchronization

The UberLogger features an internal real-time clock (RTC) that tracks time, even during power off (if the battery is installed, which it is by default). You can synchronize the RTC with your local time via the "Configuration" page. This ensures accurate timestamps for recorded data.

Note: Data is saved in Coordinated Universal Time (UTC), which helps merge datasets easily across time zones.

File options

Click to see the explanation

The File Options section allows you to configure how the UberLogger saves its data.

File Format:

  • CSV: CSV format writes data directly to CSV files, ideal for low sample rates or short measurement durations. CSV files are easily readable in tools like Excel.
  • Raw Data: Raw format records raw data for long measurements, which can be later converted into CSV files using the provided Python or Windows conversion tools. Raw data is more efficient for long recordings.

File Name Mode:

  • Sequential: Generates filenames in a sequential order (e.g., `log0`, `log1`, etc.), overwriting existing files with the same name.
  • Timestamp: Generates filenames with a timestamp in the format `YYYYMMDD_Time_FilePrefix.csv`. This avoids overwriting and helps track when files were created.

Calibration

Click to see the explanation

The Calibration section allows you to perform zero-calibration of the analog channels.

Zero calibration ensures that each analog input reads 0 when no signal is present. It is essential for accurate measurements.

You can initiate calibration by pressing the "Start Calibration" button. Ensure that all inputs are properly connected before starting calibration to get accurate results.

Import/Export Configuration

Click to see the explanation

The Import/Export Configuration section allows you to save or load the configuration settings of the UberLogger.

Export Configuration: You can download the current configuration settings to a file for backup or to apply to another device later.

Import Configuration: This allows you to load a previously saved configuration file to restore settings or replicate configurations across multiple devices.

File Browser

Click to see the explanation

The File Browser allows you to manage the data files stored on the SD card.

You can download, delete, or refresh the list of files. Formatting the SD card and unmounting it are only possible when the logger is not actively logging data.

  • Download: Click on "Download" to retrieve a file from the SD card to your local machine.
  • Delete: Select "Delete" to remove the file from the SD card.
  • Format SD-card: This option will erase all contents from the SD card and prepare it for fresh data logging.
  • Unmount SD-card: Safely unmount the SD card to remove it without causing data corruption.
